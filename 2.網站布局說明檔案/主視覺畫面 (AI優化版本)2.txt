# 🧭 全站共用原則（交給 Cursor 前貼在最上面）

* **動畫只動 `transform/opacity`**；避免觸發 layout/paint 的屬性，確保 60FPS，效能由瀏覽器合成器處理。([web.dev][1])
* \*\*用原生捲動與黏滯定位（sticky）\*\*做場景釘住；避免 scroll-jacking（改變捲動機制的一屏一停/強制節拍）。([MDN Web Docs][2])
* \*\*捲動驅動動畫（Scroll-linked animations）\*\*作為主方法（支援度允許時），不支援時退回 JS/觀察器。([MDN Web Docs][3])
* **支援 `prefers-reduced-motion`**：偵測到「減少動態」時，全站改為淡入/靜態，不做視差與位移。([MDN Web Docs][4])
* **圖檔建議**：Hero/人物各 2 版（桌機 1600×900 ≤400KB；手機 1080×1350 ≤350KB），優先 AVIF/WebP。
* **固定 CTA**：每幕右下角保留「加 LINE 詢問體驗｜前往蝦皮下單」。

# 🎬 幕 0（導引）：「向下滑動，注入能量」

**目的**：提示捲動進場、先載入首屏。
**素材**

* `bg-stars.webp`：深藍紫星空底（柔霧面）。
* `hint.txt`：右下角小字「向下滑動，注入能量」。
  **動效**
* 背景星點**微亮度呼吸**（opacity 0.9→1.0）。
* 提示字**淡入→脈動**（opacity、scale 0.98→1.0）。
  **出場節點（相對本幕進度）**
* 0–20%：星點淡入；20–100%：提示字輕微脈動。
  **降級策略**
* `prefers-reduced-motion: reduce`：僅顯示靜態星空＋固定提示字。([MDN Web Docs][4])

# 🌌 幕 1（宇宙能量匯聚 → 灌注眼鏡）

**目的**：建立世界觀，將能量匯聚至中心眼鏡。
**素材**

* `bg-nebula.webp`：藍紫能量雲（無接縫）。
* `fg-gold-particles.webp`：金色粒子雲（透明背景，2 層：前/後景）。
* `obj-glasses.png`：**去背**能量眼鏡，置中。
  **動效**
* `obj-glasses`：**position: sticky** 置中，不位移。([MDN Web Docs][2])
* `fg-gold-particles`：**前後景微差速**朝中心做 `translateY/translateZ` 位移（幅度≤20%）；同時微量 `opacity` 提升呈「灌注」。([web.dev][5])
* `bg-nebula`：極細幅度視差（位移≤10%）。
  **出場節點**
* 0–30%：星雲淡入；
* 30–80%：金色粒子由四周→中心（前景位移量 > 後景）；
* 80–100%：粒子靠近鏡框邊緣，亮度略增。
  **降級策略**
* 減少動態：取消視差與位移，改為**一次性淡入**三層素材。([MDN Web Docs][4])
* 低階裝置：粒子前後景只保留**單層**，避免「層爆炸」。可用 `will-change: transform` 但避免過度提升層級。([web.dev][6])
